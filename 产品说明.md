# Rune 产品说明

## 产品定位

Rune 是一个面向脚本和命令行工具的小型桌面辅助程序。它让 shell 脚本可以快速弹出轻量 GUI 窗口，用来做确认、提示、输入、结构化表单填写，以及实时日志查看。

它适合不想开发完整桌面应用，但又需要比纯终端交互更清晰、更友好的场景。例如发布脚本、构建脚本、运维检查脚本、自动化任务、内部工具等。

## 核心能力

- 弹出确认窗口：用于让用户确认是否继续执行危险或重要操作。
- 弹出提示窗口：用于展示构建完成、任务失败、提醒信息等。
- 读取单个输入值：用于输入分支名、用户名、路径、备注等简单内容。
- 渲染结构化表单：通过 TOML、JSON、stdin 或 CLI 参数定义多个表单项。
- 展示实时输出流：把 stdin 或子进程输出显示在桌面窗口中，适合查看日志。
- 输出 JSON：表单提交后向 stdout 输出一行 JSON，方便脚本继续处理。
- 支持样式配置：可以通过全局配置或单次 `--style` 调整主题、字号、控件高度等。

## 典型使用场景

### 发布前确认

发布脚本可以先弹出确认窗口，避免误操作：

```bash
rune confirm "Deploy to production?"
```

用户点击确认后，脚本继续执行；用户取消时，命令返回取消退出码。

### 构建完成提示

长时间构建结束后，可以弹出提示：

```bash
rune alert "Build finished"
```

### 获取一个输入值

脚本可以读取用户输入，并把结果作为普通文本输出：

```bash
branch=$(rune input "Branch name" --default main)
```

### 运行结构化表单

复杂一些的任务可以用配置文件描述表单：

```bash
rune form --config examples/demo-form.toml
```

提交后，Rune 会输出一行 JSON，例如：

```json
{"name":"demo","notes":"这里可以填写多行内容。","environment":"staging","confirmed":true}
```

### 查看命令输出

可以把子进程输出显示在窗口里：

```bash
rune stream -- cargo test
```

也可以把管道输入传给 Rune：

```bash
cargo test 2>&1 | rune stream --title "Tests"
```

## 命令概览

```text
rune form [OPTIONS]
rune stream [OPTIONS] [-- CMD...]
rune confirm <TEXT> [OPTIONS]
rune alert <TEXT> [OPTIONS]
rune input <TEXT> [OPTIONS]
rune config <init|show|path>
```

常用窗口参数：

```text
--title <TITLE>
--style <FILE>
--width <PX>
--always-on-top
--theme <light|dark|system|custom>
--timeout <SECS>
```

## 表单配置

`rune form --config` 使用当前任务的表单配置。配置文件可以是 TOML 或 JSON。TOML 示例：

```toml
title = "Deploy"
show_header_title = false
width = 520
control_width = "full"
control_height = 48
submit_label = "Deploy"
cancel_label = "Cancel"

[[items]]
type = "markdown"
content = "## Deploy checklist\nReview before continuing."

[[items]]
type = "input"
id = "branch"
label = "Branch"
default = "main"
required = true

[[items]]
type = "select"
id = "environment"
label = "Environment"
options = ["dev", "staging", "prod"]
default = "staging"

[[items]]
type = "checkbox"
id = "confirmed"
label = "I have reviewed the deployment"
required = true
```

支持的表单项：

- `text`：普通提示文字。
- `markdown`：Markdown 内容块。
- `input`：单行输入框。
- `search`：带按钮的搜索输入框，点击按钮时会输出一次搜索事件 JSON。
- `textarea`：多行输入框。
- `select`：下拉选择。
- `checkbox`：复选框。

常用顶层字段：

- `title`：窗口标题。
- `show_header_title`：是否在窗口内容顶部显示大标题。
- `width`：窗口宽度。
- `control_width`：默认控件宽度，可用 `"full"`、像素数字、百分比字符串，例如 `"75%"`。
- `control_height`：当前表单的单行控件高度。
- `submit_label`：提交按钮文字。
- `cancel_label`：取消按钮文字。

## 样式配置

Rune 默认读取全局样式配置。也可以用 `--style <FILE>` 为单次命令指定样式文件。

默认全局配置路径：

- macOS/Linux：`$XDG_CONFIG_HOME/rune/config.toml`
- macOS/Linux 备用路径：`~/.config/rune/config.toml`
- Windows：`%APPDATA%/rune/config.toml`

创建默认配置：

```bash
rune config init
```

查看当前配置：

```bash
rune config show
```

查看配置路径：

```bash
rune config path
```

样式文件示例：

```toml
[window]
theme = "dark"
always_on_top = false
corner_radius = 8
shadow = true

[window.header]
height = 40
font_size = 16
show_icon = true

[window.body]
padding = 16
font_size = 14
line_height = 1.5
control_height = 36

[window.colors]
background = "#1e1e1e"
text = "#e0e0e0"
accent = "#5b8def"
border = "#333333"
```

`--config` 和 `--style` 的职责不同：

- `--config` 描述当前表单或当前任务，例如标题、宽度、表单项、控件宽度、控件高度。
- `--style` 描述默认视觉风格，例如主题、内边距、字号、默认控件高度。

当表单配置里写了 `control_height` 时，它会覆盖样式配置里的默认控件高度，只影响这一次表单。

## 输出和退出码

表单提交成功时，Rune 向 stdout 输出一行 JSON。取消、超时、配置错误等情况通过退出码表达。

```text
0    成功
1    用户取消
2    配置或解析错误
5    超时
130  被中止
```

脚本可以根据退出码判断下一步：

```bash
if rune confirm "Continue?"; then
  echo "confirmed"
else
  echo "cancelled"
fi
```

## 安装和开发

本地安装：

```bash
cargo install --path .
```

开发时可以直接通过 Cargo 运行：

```bash
cargo run -- form --text "Hello from Rune"
```

提交前建议运行：

```bash
cargo fmt
cargo test
git diff --check
```

## 设计边界

Rune 不是完整的应用框架，也不负责复杂业务逻辑。它的重点是让脚本快速获得一个桌面交互界面，并把结果稳定地交还给脚本。

如果需要复杂状态管理、多页面导航、数据库、用户账号系统或长期运行的完整 GUI 应用，应使用专门的桌面应用框架。Rune 更适合作为命令行工具和自动化脚本的轻量交互层。
